# Himura Cache

Высокопроизводительный кэш-сервис на Rust, совместимый с протоколом RESP (Redis Serialization Protocol).

## Возможности

- **RESP протокол** — полная совместимость с Redis клиентами
- **LRU вытеснение** — с точным отслеживанием памяти
- **AOF лог** — персистентность с асинхронной записью
- **Асинхронный TCP сервер** — обработка тысяч одновременных подключений
- **Конфигурация TOML** — гибкие настройки сервера, кэша и AOF
- **Шардирование** — автоматическое включение для кэшей >100MB

## Быстрый старт

### Установка

```bash
cargo build --release
```

### Запуск

```bash
cargo run --release
```

### Подключение через redis-cli

```bash
redis-cli -h 127.0.0.1 -p 6379
```

Примеры команд:
```
PING
SET key1 value1
GET key1
DEL key1
INFO
FLUSHALL
EXPIRE key1 10
TTL key1
SETEX key1 60 value1
```

## Конфигурация

Файл `config.toml` в корневой директории:

```toml
[server]
host = "127.0.0.1"
port = 6379
max_connections = 10000

[cache]
max_memory = "100MB"

[aof]
enabled = true
path = "./data/aof.log"
```

### Параметры конфигурации

#### [server]
| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| host | "127.0.0.1" | Адрес для прослушивания |
| port | 6379 | Порт для прослушивания |
| max_connections | 10000 | Максимальное количество подключений |

#### [cache]
| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| max_memory | "100MB" | Максимальный объем памяти (KB, MB, GB) |

#### [aof]
| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| enabled | true | Включить AOF логирование |
| path | "./data/aof.log" | Путь к файлу AOF лога |

## Поддерживаемые команды

| Команда | Описание |
|---------|----------|
| PING [message] | Проверка доступности |
| GET key | Получение значения |
| SET key value | Установка значения |
| DEL key [key...] | Удаление ключей |
| INFO | Статистика сервера |
| FLUSHALL | Очистка всего кэша |
| EXPIRE key ttl | Установка TTL в секундах |
| TTL key | Получение оставшегося TTL |
| SETEX key ttl value | SET с TTL |

## Архитектура

```
src/
├── main.rs           # Точка входа
├── lib.rs            # Публичный API
├── server.rs         # TCP сервер
├── cache.rs          # LRU кэш
├── sharded_cache.rs  # Шардированный кэш
├── command.rs        # Обработчики команд
├── resp.rs           # RESP парсер/сериализатор
├── config.rs         # TOML конфигурация
└── aof.rs            # AOF логирование
```

## Производительность

- Наносекундная задержка на операциях GET/SET
- Поддержка 10000+ одновременных подключений
- Шардированный кэш для лучшей масштабируемости
- Оптимизация малых значений (inline до 24 байт)
